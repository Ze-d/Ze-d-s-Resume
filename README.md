## 项目经历

### ZCLI：具备长期记忆与任务调度的 Coding Agent

**技术栈：** Python, Anthropic SDK, Pytest, Git Worktree, CLI

**背景：**
为解决现有 Coding Agent 在长周期、多文件重构任务中容易丢失上下文、无法处理子任务依赖以及缺乏安全沙箱的痛点，我参考 Claude Code 的设计哲学，从零构建了一个生产级 Coding Agent。

**主要工作：**

1. **分层上下文压缩机制**
   为应对长时间 Agent 会话和大型工具输出导致的 Token 超限难题，设计了一种混合压缩方案。在权衡“纯摘要”的高失真和“纯裁剪”的信息丢失后，通过大结果磁盘寻址、近期历史 LLM 慢蒸馏、旧工具结果元数据裁剪等方式，在保留关键决策链的前提下，将长任务 Token 消耗降低约 60%，使 Agent 能连续处理 200+ 步复杂任务而不失忆。

2. **健壮的内存与会话系统**
   针对网络抖动导致工具调用失败并污染 Agent 记忆链的问题，实现了异常调用识别与回溯修复算法。该机制能在下次循环中自动剔除无效历史，确保 Agent 在遭遇故障后能够恢复并继续原有任务，避免上下文级联污染。

3. **支持依赖的持久化任务图**
   为使 Agent 能处理带依赖关系的复杂任务，基于图结构设计了持久化 Task Graph。通过严格的 DAG 状态流转控制，从机制上避免子任务死锁。同时将 Task Node 与 Git Worktree 绑定，实现任务维度的代码沙箱隔离，为后续并行 Subagent 协同提供支持。

### ScholarForge：基于多智能体的深度研究 Agent

**技术栈：** Python, LangGraph, LangChain, MCP, Tavily, RAG

**背景：**
为解决单 Agent 在复杂学术文献调研中搜索面窄、易产生认知偏差的问题，我构建了一个利用本地 Zotero 论文库和网络搜索结果进行交叉验证的深度研究 Agent。

**主要工作：**

1. **Supervisor-Worker 多智能体架构**
   在对比链式 ReAct 和群组辩论两种方式后，选择 Supervisor-Worker 架构以平衡多样性和效率。由 Supervisor 动态拆分查询，并行调度多个 Researcher，在验证数据集上将零样本复杂查询的结果覆盖率从 70% 提升到 92%。

2. **高鲁棒性的 LangGraph 工作流**
   为解决长链路研究因单点失败而整体作废的问题，利用 LangGraph 的状态图特性，将研究流程建模为可中断、可恢复、可回溯的有状态图，提升研究过程的工程鲁棒性，使其具备应对网络波动和 LLM 限流的韧性。

3. **本地-网络知识交叉验证**
   通过 MCP 协议集成 Zotero 本地文献库，实现本地论文知识图谱与 Tavily 网络搜索结果的交叉验证，确保生成报告兼具学术深度与信息时效性。
